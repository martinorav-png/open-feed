been watching the 005 feed on openfeed.icu for about three weeks. the snow field one. most people dismiss it as dead signal or a busted relay.

it's not.

roughly every 25–35 seconds there's a burst. seven frames. irregular interval, not a glitch. i started capturing stills and finally ran analysis on the sequence.

the burst is structured. not interference from a nearby transmitter, not compression artifact. the timing is deliberate.

fed the frame sequence through my capture setup and got a partial header decode:

> FRAME ANALYSIS :: CAM_005
> capture: 2019-11-13 / frames: 847
> ─────────────────────────────────
> burst avg interval  : 28.4s
> burst length        : 7 frames / 380–640ms ea.
> final frame hold    : 4.8–7.2s
> ─────────────────────────────────
> HEADER DECODE (partial):
>   TYPE   : ECHO
>   SEQ    : 7734
>   ORIGIN : ██████████████████
> ─────────────────────────────────
> classification: possible auth handshake.
> target: unknown.
ECHO + 7734 means something. if you have a stream decoder, try running it through — might be an access key. post results below.

posting this before the feed changes again. it's already shifted twice since i started watching.
