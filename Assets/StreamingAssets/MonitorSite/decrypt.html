been watching the 005 feed on openfeed.icu for about three weeks. the snow field one. most people think it's dead signal or a busted relay.

it's not.

roughly every 25–35 seconds there's a burst. seven frames. irregular interval. i started capturing stills and finally had time to run analysis on them.

the burst is structured. this is not interference from a nearby transmitter and it's not compression artifact. the timing is deliberate.

ran the frame sequence through my capture setup and got a partial header decode:

> FRAME ANALYSIS :: CAM_005
> capture: 2019-11-13 / frames: 847
> ────────────────────────────────
> burst avg interval : 28.4s
> burst length       : 7 frames / 380–640ms ea.
> final frame hold   : 4.8–7.2s
> ────────────────────────────────
> HEADER DECODE (partial):
>   TYPE   : ECHO
>   SEQ    : 7734
>   ORIGIN : ██████████████████
> ────────────────────────────────
> possible auth handshake. target: unknown.
ECHO + 7734 means something. if you have a StreamDecrypt build, try running it through — might be a stream access key. the stream id is probably from the feed index.

posting here before this disappears. the feed has already changed twice since i started watching.
